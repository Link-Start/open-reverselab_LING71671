# Digital Goods Inventory Patterns — 数字商品库存泄露模式

> 自动发卡/数字交付平台的常见数据库泄露模式——PHP 流程控制缺陷、归属校验缺失、数据聚合泄露。关注"为什么发生"而非"如何在某系统上操作"。

## 关键词

`数字商品` `库存泄露` `PHP流程缺陷` `校验缺失` `数据聚合` `归属校验` `CDK` `渲染消毒`

## 1. 流程控制缺陷

校验失败后未终止执行流，导致后续数据库操作继续运行并输出不应暴露的数据。

```
校验 → 输出错误信息 → 缺少 die/return → 继续执行 SQL → 全量输出
```

**解决**：每个错误分支显式终止。使用中间件统一处理。

## 2. 聚合查询泄露

查询条件为空/零值时，SQL 可能匹配全表而非"无匹配"。

```sql
-- 预期：WHERE order_id = NULL → 0 rows
-- 缺陷：某些 ORM/拼接将空值解释为 WHERE 1=1
```

**解决**：空值显式拒绝；参数化查询前校验有效性。

## 3. 归属校验缺失

列表接口仅校验 CSRF 头，不校验登录态和资源归属，可遍历全量记录。

**解决**：列表/详情接口强制登录 + 归属过滤。
